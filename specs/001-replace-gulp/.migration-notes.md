## Current Gulp Implementation

### Gulpfile.js Structure

The current gulpfile.js (37 lines) defines three tasks that run sequentially:

```javascript
'use strict';

var {spawn} = require('child_process');
var gulp = require('gulp');
var docs = require('./scripts/docs.js');

gulp.task('quality', function(cb) {
    const cmd = spawn('npx', [
        'eslint',
        '*.js',
        '{rules,test,scripts}/**/*.js'
    ], {stdio: 'inherit', shell: true});
    cmd.on('close', cb);
});

gulp.task('test', function(cb) {
    const cmd = spawn('npx', [
        'nyc',
        '--reporter=lcov',
        '--reporter=text',
        'mocha',
        'test/**'
    ], {stdio: 'inherit', shell: true});
    cmd.on('close', cb);
});

gulp.task('docs', function(done) {
    docs.updateReadme('README.md');
    docs.createDocFiles();
    // docs.testDocs(cb);
    done();
});

gulp.task('default', gulp.series('quality', 'docs', 'test'));
```

### Task Behavior

**1. Quality Task** (`gulp quality`):
- **Command**: `npx eslint "*.js" "{rules,test,scripts}/**/*.js"`
- **Purpose**: Lint all JavaScript files in project root and subdirectories
- **File Patterns**:
  - `*.js` - All JS files in project root
  - `{rules,test,scripts}/**/*.js` - All JS files in these subdirectories
- **Output**: ESLint violations with ANSI colors
- **Exit Codes**: 0 on success, non-zero on lint errors
- **stdio Handling**: `stdio: 'inherit'` - streams output directly
- **Execution Time**: ~8.73 seconds

**2. Docs Task** (`gulp docs`):
- **Command**: Calls `scripts/docs.js` functions directly
- **Purpose**: Generate/update project documentation
- **Operations**:
  - `docs.updateReadme('README.md')` - Updates README with rule info
  - `docs.createDocFiles()` - Creates individual doc files in docs/
- **Output**: Minimal (silent on success)
- **Exit Codes**: 0 on success (calls `done()` callback)
- **Execution Time**: ~49 milliseconds

**3. Test Task** (`gulp test`):
- **Command**: `npx nyc --reporter=lcov --reporter=text mocha test/**`
- **Purpose**: Run Mocha tests with NYC coverage reporting
- **Test Framework**: Mocha
- **Coverage Tool**: NYC (Istanbul)
- **Coverage Formats**:
  - LCOV (for CI/CD integration) - written to coverage/lcov.info
  - Text (console output) - displays coverage table
- **Test Count**: 1470 tests (as of baseline)
- **Output**: Test results with colors, progress, coverage table
- **Exit Codes**: 0 if all tests pass, non-zero on failures
- **stdio Handling**: `stdio: 'inherit'` - streams output directly
- **Execution Time**: ~1 second (after docs)

**4. Default Task** (`gulp` or `gulp default`):
- **Command**: Sequential execution of quality → docs → test
- **Pipeline**: `gulp.series('quality', 'docs', 'test')`
- **Fail-Fast**: Yes - stops on first task failure
- **Total Execution Time**: ~10.3 seconds
- **Invoked By**: `npm test` command

### stdio: 'inherit' Behavior

All tasks use `stdio: 'inherit'` in child_process.spawn():
- **stdout**: Streams directly to parent process
- **stderr**: Streams directly to parent process
- **stdin**: Inherited from parent process
- **Effect**: Real-time output, ANSI colors preserved, no buffering

### Exit Code Handling

- Gulp uses `cmd.on('close', cb)` to propagate exit codes
- Non-zero exit codes from child processes cause Gulp to fail
- Sequential tasks (gulp.series) stop on first failure
- npm test exits with same code as Gulp

## npm Scripts Equivalent

### Proposed package.json scripts:
```json
{
  "scripts": {
    "lint": "eslint \"*.js\" \"{rules,test,scripts}/**/*.js\"",
    "test:run": "nyc --reporter=lcov --reporter=text mocha test/**",
    "docs": "node scripts/docs.js",
    "test": "npm run lint && npm run docs && npm run test:run"
  }
}
```

### Key Differences from Gulp:
- **No spawn wrapper**: Commands run directly via npm
- **stdio inheritance**: Automatic (npm scripts default behavior)
- **Sequencing**: `&&` operator provides fail-fast
- **Exit codes**: Automatically propagated
- **Overhead**: Minimal (~10-20ms vs Gulp's ~500ms)
